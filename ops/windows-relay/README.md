# FunPay Home Relay for Windows

Лёгкий клиент использует только встроенные `ssh.exe`, `ssh-keygen.exe`,
PowerShell и Task Scheduler. Браузер на домашнем ПК не запускается. Клиент
создаёт обратный dynamic SOCKS-туннель:

```text
backend -> home-relay:1080 -> isolated SSH sidecar -> Windows PC -> Internet
```

## Установка

Откройте мастер **Настроить домашний прокси** и вставьте сгенерированную им
команду PowerShell. Команда проверяет SHA-256 архива до распаковки. Для
автозапуска она до скачивания проверяет права администратора, затем создаёт
защищённую временную папку с ACL только для `SYSTEM` и администраторов и запускает
проверенный установщик через Windows PowerShell 5.1.

Ручной запуск распакованного ZIP допустим только без повышения прав и только
если Windows OpenSSH Client уже установлен:

```powershell
.\Install.ps1 `
  -PairingUrl "https://funpay-bot.duckdns.org/api/proxy-routes/home-relay/enroll" `
  -PairingCode "<одноразовый-код>" `
  -DisableAutoStart `
  -CreateDesktopShortcuts
```

Без `-PairingCode` установщик безопасно запросит код интерактивно. Повторный
запуск идемпотентно обновляет
скрипты и ярлыки, не меняя ключ. Если повторный запуск содержит новые
`-PairingUrl` и `-PairingCode`, установщик повторно регистрирует существующий
public key, обновляет route и pinned SSH host key, проверяет tunnel и только
затем возобновляет работу. Сам setup token не записывается. Это позволяет
восстановиться после удаления route или пересоздания server-side SSH-sidecar.

Переход с уже установленного ручного режима на `-EnableAutoStart` останавливает
старый tunnel, регистрирует новый локальный ключ и только после успешной проверки
запускает защищённую системную копию. Прежний каталог остаётся неактивным
backup, поэтому одновременно два forward на `1080` не работают.

Ручная установка не требует прав администратора, если Windows OpenSSH Client
уже установлен. Если его нет, первый административный проход устанавливает
только системный компонент и завершает работу без привязки; ту же ещё действующую
команду нужно повторить в обычном PowerShell. Для `-EnableAutoStart` и ремонта
boot-установки откройте PowerShell через **«Запуск от имени администратора»** и
вставьте команду из админки. Установщик намеренно не повышает права сам и
отказывается исполнять административную установку из доступной пользователю
папки.

При переходе из ручного режима в автозапуск старые процессы останавливаются без
исполнения файлов из `%LOCALAPPDATA%`. Старый каталог остаётся как неактивная
резервная копия; после проверки boot-режима его можно удалить из обычного
PowerShell.

## Управление

- `Start.ps1` — ручной запуск;
- `Stop.ps1` — ручная остановка;
- `Status.ps1` — локальное состояние и статус Task Scheduler;
- `Uninstall.ps1 -Force` — удалить локальный ключ, задачу и runtime.

В ручном режиме установщик создаёт ярлыки Start/Stop/Status в меню «Пуск» и, по
выбору, на рабочем столе. В boot-режиме он намеренно не пишет с повышенными
правами в доступные пользователю Start Menu/Desktop: системная задача запускает
relay автоматически, а ручное управление выполняется защищёнными скриптами из
`%ProgramData%\FunPayHomeRelay` в PowerShell от имени администратора.
Защищённые Start/Stop/Status/Uninstall совместимы с Windows PowerShell 5.1 и
PowerShell 7: перед работой с Task Scheduler они оставляют в `PSModulePath`
только системный каталог модулей Windows.
Переключение уже установленного boot-режима на `-DisableAutoStart` отключает
задачу, но сохраняет защищённый ProgramData runtime; это ручной режим с UAC, а
не перенос ключа обратно в доступный пользователю каталог.

При включённой автозагрузке задача запускается от `SYSTEM` **при старте Windows,
до входа пользователя** и восстанавливает соединение с bounded backoff. ПК
должен быть включён, интернет должен работать, а сон/гибернация должны быть
отключены на время проверок.

## Безопасность

- приватный Ed25519-ключ генерируется на ПК и никогда его не покидает;
- одноразовый pairing code передаётся только по HTTPS и не записывается в
  relay-конфиг или логи. Готовая команда может остаться в истории PowerShell,
  поэтому её нельзя пересылать; после первого успешного enrollment или TTL код
  больше не действует;
- runtime разрешает только public-key auth и использует pinned `known_hosts`;
- если API не вернул host key, `accept-new` разрешён ровно одному install-probe;
  все дальнейшие подключения используют `StrictHostKeyChecking=yes`;
- tunnel создаётся с `ExitOnForwardFailure`, keepalive и без shell/PTY;
- ручной режим хранится в `%LOCALAPPDATA%\FunPayHomeRelay` с доступом только
  текущего Windows SID и `SYSTEM`;
- boot-режим хранится в `%ProgramData%\FunPayHomeRelay`: runtime доступен
  пользователям только на чтение, а приватный ключ — только `SYSTEM` и группе
  локальных администраторов; обычный пользователь не может подменить код,
  запускаемый системной задачей;
- логи ротируются, pairing code, Authorization, token и password редактируются;
- при недоступности relay backend обязан завершить вход ошибкой и **не**
  переключаться на IP VPS.

`0.0.0.0:1080` относится только к сетевому namespace SSH-sidecar. Порт 1080
никогда не публикуется на хосте (`ports:`/firewall запрещены). Наружу доступен
только SSH `2222/tcp`. Подробнее о серверном контракте —
[`docs/windows-home-relay.md`](../../docs/windows-home-relay.md).
